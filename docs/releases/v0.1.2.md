## OpenCode Snapshot Tool v0.1.2

### Cleanup correctness and storage clarity

- Fixes **Clean now** so trees explicitly released in Preview are pruned immediately instead of receiving an unintended second retention grace period.
- Measures cleanup before/after bytes across both processed and untouched repositories, avoiding overstated reclaimed-space results.
- Adds an integration regression test that verifies a released Git object disappears and the snapshot repository actually shrinks with the default seven-day retention setting.
- Clarifies that Snapshot Storage is the actual content under the snapshot root.
- Renames the direct LFS/temp-file estimate so it is not confused with Git pack savings, which are measured after cleanup.
- Adds the application screenshot to the README.

Download `OpenCode-Snapshot-Tool-v0.1.2-windows-x64.zip`, extract it, and run `opencode-snapshot-tool.exe`.
