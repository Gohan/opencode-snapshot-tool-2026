## OpenCode Snapshot Tool v0.1.3

### Repository storage analysis and process-aware cleanup

- Adds a repository storage detail viewer with drill-down pages for paths, file types, Git objects, snapshots, and reclaim options.
- Adds deep analysis that attributes packed blobs to current-tree paths and separates current state, retained history, and unprotected objects.
- Maps each running OpenCode PID to its worktree and exact snapshot store so unrelated OpenCode instances can remain open during cleanup.
- Adds guarded per-project Safe GC, current-state-only history reset, and full snapshot-store purge flows, each with a dedicated preview and confirmation.
- Rechecks processes, Git locks, and store write activity immediately before destructive execution.
- Improves migrated project/store mapping and distinguishes active, inactive, and possibly active repositories.
- Adds the FC-style cleaning-machine application icon with a transparent background for the custom title bar, taskbar, and Windows executable.
- Expands automated coverage to 20 tests for storage accounting, deep analysis, process mapping, cleanup, reset, purge, and OpenCode reinitialization.

Download `OpenCode-Snapshot-Tool-v0.1.3-windows-x64.zip`, extract it, and run `opencode-snapshot-tool.exe`.

SHA-256: `50d48cce381eae2dac19448d40c8df8aef10f9e0652139e74ee487c6944c8633`
